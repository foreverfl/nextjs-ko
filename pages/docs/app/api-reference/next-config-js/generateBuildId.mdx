# generateBuildId

{/* 이 문서의 내용은 app 라우터와 pages 라우터 간에 공유됩니다. <PagesOnly>내용</PagesOnly> 컴포넌트를 사용하여 Pages 라우터에만 해당하는 내용을 추가할 수 있습니다. 공유된 내용은 컴포넌트로 감싸지 않아야 합니다. */}

Next.js는 `next build` 중에 애플리케이션의 어떤 버전이 제공되고 있는지 식별하기 위해 ID를 생성합니다. 동일한 빌드를 사용하여 여러 컨테이너를 부팅해야 합니다.

환경의 각 단계마다 다시 빌드하는 경우, 컨테이너 간에 사용할 일관된 빌드 ID를 생성해야 합니다. `next.config.js`에서 `generateBuildId` 명령을 사용하세요:

```jsx filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // 이는 무엇이든 될 수 있으며, 최신 git 해시를 사용합니다
    return process.env.GIT_HASH;
  },
};
```
