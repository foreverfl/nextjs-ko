# httpAgentOptions

{/* 이 문서의 내용은 app 라우터와 pages 라우터 간에 공유됩니다. `<PagesOnly>Content</PagesOnly>` 컴포넌트를 사용하여 Pages 라우터에만 특정한 내용을 추가할 수 있습니다. 공유된 내용은 컴포넌트로 감싸지 않아야 합니다. */}

Node.js 18 버전 이전에는 Next.js가 자동으로 `fetch()`를 [undici](/docs/architecture/supported-browsers#polyfills)로 폴리필하고 [HTTP Keep-Alive](https://developer.mozilla.org/docs/Web/HTTP/Headers/Keep-Alive)를 기본적으로 활성화합니다.

서버 사이드의 모든 `fetch()` 호출에 대해 HTTP Keep-Alive를 비활성화하려면 `next.config.js`를 열고 `httpAgentOptions` 설정을 추가하세요:

```js filename="next.config.js"
module.exports = {
  httpAgentOptions: {
    keepAlive: false,
  },
};
```
