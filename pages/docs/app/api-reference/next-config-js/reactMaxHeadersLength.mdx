# reactMaxHeadersLength

정적 렌더링 중에 React는 응답에 추가될 수 있는 헤더를 생성할 수 있습니다. 이는 브라우저가 폰트, 스크립트, 스타일시트와 같은 리소스를 미리 로드할 수 있게 함으로써 성능을 향상시키는 데 사용될 수 있습니다. 기본값은 `6000`이지만, `next.config.js`에서 `reactMaxHeadersLength` 옵션을 구성하여 이 값을 재정의할 수 있습니다:

```js filename="next.config.js"
module.exports = {
  reactMaxHeadersLength: 1000,
};
```

> **알아두면 좋은 점**: 이 옵션은 App Router에서만 사용할 수 있습니다.

브라우저와 서버 사이의 프록시 유형에 따라 헤더가 잘릴 수 있습니다. 예를 들어, 긴 헤더를 지원하지 않는 리버스 프록시를 사용하는 경우, 헤더가 잘리지 않도록 하기 위해 더 낮은 값을 설정해야 합니다.
