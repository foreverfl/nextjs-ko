# logging

{/* 이 문서의 내용은 app 라우터와 pages 라우터 간에 공유됩니다. `<PagesOnly>Content</PagesOnly>` 컴포넌트를 사용하여 Pages 라우터에만 특정한 내용을 추가할 수 있습니다. 공유된 내용은 컴포넌트로 감싸지 않아야 합니다. */}

Next.js를 개발 모드로 실행할 때 로깅 수준과 전체 URL이 콘솔에 기록되는지 여부를 구성할 수 있습니다.

현재 `logging`은 `fetch` API를 사용한 데이터 가져오기에만 적용됩니다. Next.js 내의 다른 로그에는 아직 적용되지 않습니다.

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true,
    },
  },
};
```

[서버 컴포넌트 HMR 캐시](/docs/app/api-reference/next-config-js/serverComponentsHmrCache)에서 복원된 `fetch` 요청은 기본적으로 로깅되지 않습니다. 그러나 `logging.fetches.hmrRefreshes`를 `true`로 설정하여 이를 활성화할 수 있습니다.

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true,
    },
  },
};
```

또한 `logging`을 `false`로 설정하여 개발 로깅을 비활성화할 수 있습니다.

```js filename="next.config.js"
module.exports = {
  logging: false,
};
```
