# 라우트 정의하기

> 계속하기 전에 [라우팅 기본사항](/docs/app/building-your-application/routing) 페이지를 읽어보시는 것을 추천합니다.

이 페이지에서는 Next.js 애플리케이션에서 라우트를 정의하고 구성하는 방법을 안내합니다.

## 라우트 생성하기

Next.js는 **폴더**를 사용하여 라우트를 정의하는 파일 시스템 기반 라우터를 사용합니다.

각 폴더는 **URL** 세그먼트에 매핑되는 [**라우트** 세그먼트](/docs/app/building-your-application/routing#route-segments)를 나타냅니다. [중첩 라우트](/docs/app/building-your-application/routing#nested-routes)를 만들려면 폴더를 서로 중첩하면 됩니다.

특별한 [`page.js` 파일](/docs/app/building-your-application/routing/pages)을 사용하여 라우트 세그먼트를 공개적으로 접근 가능하게 만듭니다.

  alt="라우트 정의하기"

/>

이 예시에서 `/dashboard/analytics` URL 경로는 해당하는 `page.js` 파일이 없기 때문에 공개적으로 접근할 수 없습니다. 이 폴더는 컴포넌트, 스타일시트, 이미지 또는 기타 관련 파일을 저장하는 데 사용될 수 있습니다.

> **알아두면 좋은 점**: 특수 파일에는 `.js`, `.jsx`, 또는 `.tsx` 파일 확장자를 사용할 수 있습니다.

## UI 생성하기

각 라우트 세그먼트에 대한 UI를 생성하기 위해 [특별한 파일 규칙](/docs/app/building-your-application/routing#file-conventions)이 사용됩니다. 가장 일반적인 것은 라우트에 고유한 UI를 보여주는 [페이지](/docs/app/building-your-application/routing/pages)와 여러 라우트에서 공유되는 UI를 보여주는 [레이아웃](/docs/app/building-your-application/routing/layouts-and-templates#layouts)입니다.

예를 들어, 첫 번째 페이지를 만들려면 `app` 디렉토리 안에 `page.js` 파일을 추가하고 React 컴포넌트를 내보내세요:

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>;
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>;
}
```
